# `cross_task_edges_v2/` — unified cross-task EAP edges (Pipeline B canonical)

Produced 2026-04-24. All edges in this directory come from a **single, consistent
pipeline-and-checkpoint family**: `trl.SFTTrainer` full fine-tuning saved as
HuggingFace `safetensors` directories, using the **user-selected "best"
checkpoints** at `/mnt/data1/users/sglli24/fine-tuning-project-1/fine_tuned_models/`
(Pipeline B). All EAP runs use the current post-2026-01-30 `attribute_mem.py`.

This is the canonical replacement of the mixed-pipeline `cross_task_edges/`
directory on GitHub (which silently blends Pipeline A `.pt` checkpoints for
GPT-2 / Qwen2 / Llama-3.2 with Pipeline B checkpoints for Llama-2). Use this
directory when building new overlap / circuit tables that need every cell to
come from the same pipeline.

## Layout

```
cross_task_edges_v2/
├── README.md
├── <model>_<task>_pretrained_edges.csv         # base model on each task's corrupt data
├── <model>_<task>_finetuned_edges.csv          # own-task FT: FT-on-X × data-X
├── <model>_Finetuned-<ft>_Corrupted-Data_<test>_finetuned_edges.csv  # cross-task
└── same_ft_cross_data_overlap/
    ├── <model>_same_ft_cross_data_overlap.csv    # 6x6 matrix, diag=NaN
    ├── combined_long.csv                          # all 4 models, long format
    └── overlap_edges/
        └── <model>_FT-<ft>_data-<data>_overlap.csv   # per-cell shared edges
```

`<model>` ∈ `{gpt2, llama3.2, qwen2, llama2}`
`<task>`, `<ft>`, `<test>`, `<data>` ∈ `{yelp, sst2, squad, coqa, kde4, tatoeba}`

**168 edge files** (42 per model: 6 pretrained + 6 own-task FT + 30 cross-task FT)
**+ 4 overlap matrices + 1 long-format CSV + 120 per-cell overlap-edge CSVs.**

## `same_ft_cross_data_overlap/` sub-dir

For each fine-tuned model, computes the top-400 edge overlap between:

- **A** = the model's own-task circuit (FT-on-X run on X's corrupted data)
- **B** = the same model's circuit on other-task data (FT-on-X run on Y's corrupted data, Y != X)

This measures **circuit-data invariance**: how much of a FT model's "own-task
circuit" is re-activated when the fine-tuned weights are run against a
different task's input. High overlap → the circuit is data-invariant / generic;
low overlap → the circuit is data-specific.

Per-model 6x6 matrices with diagonal = NaN (the diagonal would be trivially 100%
since A == B). 30 populated cells per model × 4 models = 120 overlaps total,
matching the 120 per-cell shared-edge CSVs in `overlap_edges/`.

Generated by `src/EAP/compute_same_ft_cross_data_overlap.py`. Re-run whenever
edge files in this directory change.
